# Banned Phrases — B2B Outreach

**Rule:** If any of these appear in a draft, delete and rewrite from scratch.

---

## Category 1: Generic Openers

| Banned | Why | Replace with |
|--------|-----|--------------|
| "I'm reaching out" | Signals form letter | Start with their specific signal instead |
| "I wanted to reach out" | Passive, waste of words | Name their pain in the first sentence |
| "Touching base" | Meaningless | State exactly what you're following up on |
| "Hope this finds you well" | Robot opener | Reference something real from research |
| "Hope you're doing well" | Filler, nothing here | Omit entirely |
| "Quick question" | Never quick | Just ask the question |
| "Just wanted to..." | Undermines authority | State the purpose directly |
| "I was wondering if..." | Passive | "Would X make sense?" |
| "Checking in" | Vague follow-up | "Following up on [specific thing]" |
| "Following up per my last email" | Passive aggressive | "Any movement on [specific topic]?" |

---

## Category 2: Feature Dumps (Pain First — Always)

| Banned | Why | Replace with |
|--------|-----|--------------|
| Opening with product features | Leads with solution before pain | Establish pain first, then map feature |
| "[Product] has X, Y, Z capabilities" | Vendor dump | "[Their pain] → we help teams like yours [outcome]" |
| "Let me tell you about [product]" | Unsolicited pitch | Only mention product after pain is established |
| "[Product] is the best at..." | Unearned claim | "[Company] achieved [metric] using [product] for [use case]" |
| Listing more than 2 capabilities | Overwhelm, not focus | Pick 1 capability most relevant to confirmed pain |

---

## Category 3: Invented or Vague Metrics

| Banned | Why | Replace with |
|--------|-----|--------------|
| "Significantly faster" | Meaningless | Specific number with source (only if verified) |
| "Huge cost savings" | Unverifiable | Name the customer + metric + use case |
| "Many companies" | Vague | Name one company with a specific outcome |
| "Most [role] choose us" | Unverified | Use an actual verified proof point |
| Any metric not found in research | Fabricated | Only numbers from research files or proof-points.md |

---

## Category 4: Corporate Buzzwords

| Banned | Always |
|--------|--------|
| "Synergy" | Yes |
| "Circle back" | Yes |
| "Move the needle" | Yes |
| "Best of breed" | Yes |
| "Paradigm shift" | Yes |
| "Leverage" (as a verb) | Cliché |
| "Low-hanging fruit" | Yes |
| "Disruptive" | Yes |
| "Innovative solution" | Everything is innovative |
| "End-to-end" | Almost always |
| "Ecosystem" (to describe your product) | Yes |

---

## Category 5: Cold Call Killers

| Banned | Why | Replace with |
|--------|-----|--------------|
| "How are you today?" | Signals cold call, triggers defense | "I'll be honest — this is a cold call." |
| "Is now a good time?" | Gives an easy out | "Have I caught you at a terrible time?" |
| "I know you're busy but..." | Self-defeating | Just get to the point |
| "I just wanted to introduce myself" | No value | State why YOU, why THEM, why NOW |
| "Our platform/solution/product" | Generic | Refer to the specific capability |
| "Take you through a quick demo" | Too fast, too presumptuous | "Would a 20-min conversation make sense first?" |
| Reading from a script (audibly) | Kills trust | Internalize the framework, adapt in real time |

---

## The "So What?" Test

Before stating any product capability, apply this test:

**Statement:** "[Your product] has [feature]."
**Ask:** "So what does that mean for them?"
**Answer must be:** A business outcome in their language.

If you cannot answer "so what?" with a business outcome → don't say it.

**Example (bad → good):**
- Bad: "We have enterprise-grade security."
- Good: "Your security team won't need a separate audit for each data system — one platform, one compliance review."

**Example (bad → good):**
- Bad: "We support multi-cloud deployments."
- Good: "When you expand to a second cloud next year, you don't rebuild your data infrastructure."
